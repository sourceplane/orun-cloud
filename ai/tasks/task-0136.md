# Task 0136 — Milestone PX2: config surface (settings, flags, secrets)

# Agent

Implementer

# Current Repo Context

PX1 merged (#299). The api-edge config facade is live at all three scopes
(probed on stage 2026-06-11); `@saas/sdk` `client.config` covers settings,
feature flags, and secret metadata including rotate/revoke. The console
Config page is the de-stubbed PX1 placeholder.

# Objective

Ship the console face of config-worker: settings, feature flags, and secrets
at org, project, and environment scope — the PX2 milestone of
`specs/epics/saas-product-experience/implementation-plan.md`.

# Architect Brief

- Product bar: **Vercel project-settings / LaunchDarkly-lite** — flags flip
  optimistically; secrets are visibly write-only.
- Free: page/tab layout per scope, value-input semantics, file layout.
- Must not: add backend routes or contracts; render secret material in any
  list/read path; bypass `@saas/sdk`.
- Invalidators: secret value visible after create/rotate dialog closes; flag
  toggle without rollback on error; the PX1 stub copy still reachable.

# Milestone Boundary

`apps/web-console-next` + its test suite only.

# Required Outcomes

1. Shared config surface (tabs: Settings / Feature flags / Secrets) mounted
   at org (settings/config), project (new page + nav), environment (detail
   page section).
2. Settings: list/create/edit with JSON-or-string value input.
3. Flags: list/create + optimistic toggle with rollback.
4. Secrets: metadata list, write-only create (password input), rotate,
   revoke via ConfirmDialog.
5. Designed empty states; cache keys scope-disambiguated; unit tests for the
   pure value/scope helpers.

# Non-Goals

Setting/flag deletion (no SDK surface); config promotion (P1); audit views.

# Acceptance Criteria

Live stage walkthrough: create/edit a setting, create+toggle a flag,
create/rotate/revoke a secret at org scope; project/env mounts render; tests
green.

# Verification

Turbo lanes + post-merge authenticated stage walkthrough; prod smoke.

# PR Creation Requirement

PR with green CI before merge.
