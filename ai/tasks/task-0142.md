# Task 0142 — Milestone IG4: token broker

# Agent

Implementer

# Objective

A tenant product exchanges its control-plane credential for a short-lived,
repo-scoped GitHub installation token (design §7): repos must be linked,
permissions ⊆ the App grant, TTL ≤ 1h, never cached, never logged, audited.
D3 default posture: owner/admin + service principals via policy action
`organization.integration.token.issue`.

# Required Outcomes

1. `POST .../integrations/github/token` with linked-repo + grant-subset +
   single-connection validation; scoped mint via GitHub access_tokens API.
2. Policy action registered (owner/admin); entitlement reuse.
3. `integration.token.issued` audit (actor, repos, permissions — never the
   token); token absent from cache and logs.
4. SDK `integrations.issueGithubToken`; CLI `sourceplane integrations github
   token --repos --permissions`.
5. "Act on GitHub from your product" recipe (worker README, webhook-verifier
   style) + react-to-events recipe.

# Acceptance Criteria

Workspace suites green; unlinked repo and grant-exceeding requests denied
with safe 412s; mint request carries exactly the scoped repo ids +
permissions; audit payload proven token-free.
