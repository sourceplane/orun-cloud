# Objective

An org admin can connect a GitHub org from the console: signed single-use
state → App install popup → edge setup ingress → installation verified as the
App → connection active. Revoke works platform-side. All gates fail closed.

# Architect Brief

- Bar: Vercel "Connect Git provider" — popup flow, designed states.
- Security invariant: `installation_id ↔ org_id` binds ONLY via our signed
  single-use state (HMAC + persisted nonce hash, TTL 10 min, consume-on-use);
  unsolicited installs are orphaned, never auto-bound.
- Free: HTML popup page styling, poll cadence, SDK method names.
- Must not: trust redirect params beyond the installation id; log tokens or
  state; widen policy roles beyond owner/admin for connect/manage.
- Failure modes: cross-org state redemption; replayed state activating
  anything; the ingress path requiring (or accepting) a bearer actor.
