# Task 0134 — Console UX polish: Vercel-grade account section + dialog a11y

Branch (session): `claude/console-org-settings-redesign-ZHZyb`
Owner: `apps/web-console-next`. Frontend; human-independent.
Spec basis: parity with modern SaaS consoles (Vercel) for the account/identity
surface and dialog accessibility.

## Why (browser-confirmed)

A behaviour review in the browser surfaced two issues:

1. **Account/identity is fragmented and unlike Vercel.** The user/account lives
   as a generic person icon in the top-right that shows *no identity* (no
   avatar / name / email); "Profile" + "Security activity" sit as a mid-list
   `ACCOUNT` nav section; and the sidebar bottom is a low-value
   `v0.0 · next-console` footer. The mobile drawer has its own account footer,
   so desktop and mobile disagree. Vercel anchors the signed-in user at the
   **bottom of the sidebar** as an avatar + name chip that opens the account
   menu.

2. **Dialogs are missing a `DialogDescription`.** The create dialogs (project,
   organization, environment, invitation) render `DialogContent` with no
   description, so Radix logs `Missing Description or aria-describedby for
   {DialogContent}` every time one opens. That's an accessibility gap and it
   trips the Next.js dev error-overlay (the "flakiness" seen when the create
   project window opens).

## Outcome ("done")

- A Vercel-style account chip is anchored at the bottom of the desktop sidebar:
  avatar (initials) + display name + email, opening a menu (Profile, Security
  activity, theme toggle, Logout). The redundant mid-list `ACCOUNT` nav section
  and the version footer are gone; the topbar account icon is desktop-hidden
  (mobile keeps it, since the sidebar is a drawer there). The mobile drawer
  footer mirrors the same actions (incl. Security activity).
- Every dialog has an accessible description (or an explicit opt-out), so the
  Radix warning no longer fires and screen readers announce the dialog purpose.

## Scope (IN)
- `components/shell/sidebar-account.tsx` (new), `sidebar.tsx`, `nav-items.ts`
  (+ test), `mobile-nav.tsx`, `topbar.tsx`, `lib/query-keys.ts`.
- `DialogDescription` on the project / org / environment / invitation dialogs.

## Scope (OUT)
