# Objective

A signed GitHub delivery POSTed to `/ingress/github/webhook` is verified over
raw bytes, persisted idempotently in the inbox, attributed installation →
connection → org by the drain, and emitted as a normalized `scm.*` (or
`integration.*` lifecycle) event into event_log exactly once — with a
console-facing delivery log and replay.

# Required Outcomes

1. api-edge: public `/ingress/github/webhook` forward (raw body streamed,
   signature headers only, rate-limited, no resolveActor).
2. Worker ingest: HMAC-SHA256 verify before parse (constant-time), body cap,
   immediate 401 on failure, `inbound_deliveries` upsert keyed by
   `X-GitHub-Delivery`, fast ack.
3. Drain (scheduled handler, cron pending slot): attribute, process
   lifecycle (uninstall → revoked, suspend/unsuspend, repo-selection,
   permissions), normalize + emit transactionally with the `emitted` mark,
   bounded retries (5) with backoff → terminal `failed` with safe reason.
4. Migration `190_integrations_delivery_attribution` (connection pointer on
   the inbox) + repo-layer support.
5. Delivery log API (`GET .../integrations/{id}/deliveries`, safe
   projection) + replay (`POST .../deliveries/{id}/replay`, re-runs from the
   persisted row — never re-trusts the wire). SDK methods.
6. Fixture-driven tests for the normalizer taxonomy and the full pipeline.

# Non-Goals

Per-project event enrichment via repo links (IG3); console activity surface
