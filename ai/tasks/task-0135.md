# Objective

Make the console keep every promise it makes: no unbranded error surface, no
native browser confirm/alert, no dead nav destination, no mutation button
without an in-flight state, and real wayfinding on every page.

# Architect Brief

- Product bar: **Vercel / Linear dashboard quality**; the user moment is "I
  hit a wrong URL / a slow path / a destructive action, and the product stays
  composed."
- Free to decide: visual design of 404/error pages, confirm-dialog copy
  patterns, breadcrumb layout, dirty-guard UX, file/module layout inside
  `apps/web-console-next`.
- Must not: change contracts, add backend routes, touch other apps, introduce
  new public console routes (PX2 owns Config), or bypass the design system.
- Invalidators even if tests pass: any reachable native `confirm()`, the
  default Next.js 404 reachable at any console path, a stub page left in nav,
  breadcrumb rendered as plain text instead of `<nav>`.
