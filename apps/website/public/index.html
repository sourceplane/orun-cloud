How it converges

From intent to production, in three moves

No imperative scripts. No snowflake pipelines. You declare the end state; orun computes the plan, runs it as CI checks, and converges on merge — in dependency order.

01

Declare

One intent file. Components, environments, policies, promotion rules.

environments:
  prod:
    CONSOLE_DOMAIN: app.orun.dev
    policies:
      requireApproval: true
02

Plan

Every PR computes a scoped plan. Each component job lands as a GitHub check.

$ orun plan
✓ api-edge          2 changes
✓ hyperdrive        1 create
→ 9 checks posted to PR #42
03

Converge

Merge applies the plan in dependency order — infra first, workers after, smoke-tested.

$ orun run
✓ supabase → hyperdrive → fleet
✓ Converged · 0 drift
Why orun

Delivery engineering, not duct tape

Plans you can review

Every change becomes an explicit, scoped plan — one job per component per environment, posted as PR checks. Nothing applies that wasn't reviewed.

Composable pipelines

Capability-tagged compositions define how a component verifies and deploys. Write the pipeline once; every worker, site, and Terraform stack reuses it.

Zero committed IDs

Templates carry @@wiring(...)@@ tokens, never resource IDs or secrets. Real values resolve from your secret manager at deploy time.

GitHub-native

PRs plan. Merges converge. OIDC roles scope every job to its repo and environment — no long-lived cloud keys in CI.

Promotion with gates

dev → stage → prod with per-environment policies: approvals, triggers, parameter overrides. The same plan, promoted deliberately.

Remote state, live runs

Jobs coordinate through a shared backend — dependency-ordered across runners, observable in real time, resumable on retry.

A principle, not a feature
No committed resource IDs. Ever.

If a value identifies infrastructure, it doesn't belong in git. orun resolves it at deploy time from the wiring manifest your infra publishes — so forks, rebrands, and new accounts converge from a clean clone.

Orun Cloud

The managed backend for orun

Point backendUrl at Orun Cloud and every plan, run, and state transition is stored, ordered, and streamed.

  • Managed remote state — no buckets to provision, no lock files to babysit.
  • Real-time run console — watch jobs converge across runners, with full history.
  • Organizations & policy — teams, roles, approvals, and audit built in.
